Media queries

First things first, offkit sets up 4 variables used in style.less for the media queries. Feel free to change them to suit your needs and remember that Offkit is “mobile first”, so _base.less correspond to 0px and higher. Defaults are:

Default media queries
{% highlight css %} @mq-phone-ldscp : 30em; /* 30 * 16 = 480px */ @mq-tablet-port : 48em; /* 48 * 16 = 768px */ @mq-tablet-ldscp : 64em; /* 64 * 16 = 1024px */ @mq-desktp : 77.5em; /* 77.5 * 16 = 1240px */ {% endhighlight %}

Font settings

We first sets up our font family:

Default Font family sets
{% highlight css %} @sans-serif : 'helvetica Neue', helvetica, arial, sans-serif; /* used for headings */ @serif : 'times', 'times new roman', 'georgia'; /* used for paragraphs */ @mono : monaco; {% endhighlight %}

And this is where it gets a little more tricky. Offkit has the ability to calculate automaticaly a typographic scale (defining sizes for h1 to h6, p and two options for smaller sizes) using a preset ratio or wichever you want. It does that in one of the three units you can choose (px,em or rem). So for this to work, you must first choose the unit you'll use and a base size (in pixel) which will correspond to the p and h6 sizes (this basesize will be automatically converted in the proper unit in the next lines of the file).

Basics font settings
{% highlight css %} @font-unit : rem; /* acceptedd values [px, em, rem] */ @basesize :20; /* set in pixels */ {% endhighlight %}

We can now choose our ratio for calculating the rest of the typographic scale thanks to the @typescale-ratio. This accept the presets ratio founded in _constants.less, any kind of number you wan to use or the keywords "none" or false if you want to set everything by hand.

Accepted values for @typescale-ratio
{% highlight css %} "none", false, any number, @golden-ratio, @augmented-fourth, @major-second, @major-third, @major-sixth, @major-seventh, @major-tenth, @minor-second, @minor-third, @minor-sixth, @minor-seventh, @octave, @perfect-fourth, @perfect-fifth {% endhighlight %}

a good ressource about this one: type-scale.com.

Even after having this ratio set up, you can manualy override the calculated values in the next lines (according to the unit you provided earlier). If you are fine with the generated ones, you can just leave everything to false

Manually setting the typographic scale
{% highlight css %} @manual-size-g : false; /* not directly in use, call it with the mixin .size-g(); */ @manual-size-f : false; /* h1 -- call it with the mixin .size-f(); */ @manual-size-e : false; /* h2 -- call it with the mixin .size-e(); */ @manual-size-d : false; /* h3 -- call it with the mixin .size-d(); */ @manual-size-c : false; /* h4 (equivalent h4 / h5) -- call it with the mixin .size-c(); */ @manual-size-b : false; /* h5 -- call it with the mixin .size-b(); */ @manual-size-a : @fontsize; /* p and h6 -- call it with the mixin .size-a(); */ @manual-size-small : false; /* small size -- call it with the mixin .size-small(); */ @manual-size-smaller : false; /* smaller -- call it with the mixin .size-smaller(); */ {% endhighlight %}

Next comes the usual setting for baselin, weight and letter spacing (word spacing is a little more tricky and requieres a more specific approche, so it's up to you)

And finally, you can set up a default style for your paragraphs and generate classes for a few other build in styles like indented, dropcaps, separated by a ruler or even using a pilcrow. The default is obviously leading.

See more about that in the type section.

Grid settings

offkit allows you to perconalize your markup on a few things; grids are one of them. A set of variable gives you the opportunity to choose your own selector name for columns, rows & offsets if you use them, & wrapper:

Default grid settings
{% highlight css %} @container-name : col; /* define column class name */ @wrap-name : wrap; /* define column container (wrap) class name */ @row-name : row; /* define column container (row) class name */ @offset-name : offset; /* define offset column class name */ @offset-grid : true; /* do we want .offset-@index or not ? */ @use-wrap : true; /* use wrap style grid (.wrap > .col-1.first + .col-11) */ @use-row : true; /* use row style grid (.container (can be wrap) > .row > .col-1 + .col-11) */ {% endhighlight %}

See more about that in the grid section.

Margin settings

in this section you'll be able to set the size for margins and paddings applied to the font, the buttons, forms… etc. There are basically 3 variables (top, bottom, indent) build on top of the @line-height variable and wich are used to calculate the other measurement (you can of course personalised everything)

Default margins and paddings
{% highlight css %} @margin-top: 0; @margin-bottom: @lineheight * 1em; @indent-value: @lineheight * 1em; /* used for paragraphs if indentation is activated and paddings */ /* margin for paragraphs */ @p-margin-top: @margin-top; @p-margin-bottom: @margin-bottom/2; /* margin for titles */ @title-margin-top: @margin-bottom/2; @title-margin-bottom: @margin-bottom/3; /* for forms input */ @input-margin-top: 0; @input-margin-bottom: @margin-bottom; @input-padding: @margin-bottom/4 @indent-value/2; @label-margin-bottom: @margin-bottom/4; /* for button & submit */ @btn-margin-top: 0; @btn-margin-bottom: @margin-bottom; @btn-padding: @margin-bottom/3 @margin-bottom/2; @btn-padding-small: @margin-bottom/4 @indent-value/3; @btn-padding-large: @margin-bottom/2 @indent-value; /* for tables */ @table-margin-top: 0; @table-margin-bottom: @margin-bottom; @table-padding: @margin-bottom/4 @indent-value/2; {% endhighlight %}

Misc settings

The “Misc settings” section comes to complete that. First, a @rounded-bool defined if ou want border-radius applied to inputs, buttons, tables… etc and a @rounded-value let's you defined this default radius. After that, a few other variables let's you defined the default values for transitions and bow shadows mixins.

Default misc settings
{% highlight css %} @rounded-bool : true; @rounded-value : 2px; /* could be "2px 0 3px 4px" */ /* default shadows value */ @shadow-color : @grey; @shadow-h : 0; @shadow-v : 1px; @shadow-blur : 2px; @shadow-spread : 0px; /* default values for transitions */ @transition-property : all; @transition-duration : .2s; @transition-ease : ease-in-out; {% endhighlight %}
